## 이 상황은 언제 오는가

- "다음 분기 MAU가 얼마나 될까요?" 같은 경영진/PM 질문
- 사업 계획 수립 시즌에 지표 목표값 설정 요청
- 마케팅 예산 배분 전 "이 캠페인 효과가 얼마나 될까"
- 채용/인프라 계획을 위한 성장 예측

## 제일 먼저 할 질문

1. **예측 목적이 무엇인가?**
   - 목표 설정용 (top-down이면 예측보다 목표가 먼저)
   - 리소스 계획용 (범위 추정이 중요)
   - 의사결정 참고용 (시나리오 분석)

2. **예측 가능한 지표인가?**
   - 충분한 과거 데이터가 있는가 (최소 1년, 계절성 있으면 2년+)
   - 지표를 설명하는 주요 드라이버가 알려져 있는가
   - 구조적 변화(신사업, 시장 변화)가 예정되어 있는가

3. **어떤 수준의 불확실성을 감수할 수 있는가?**
   - 단일 숫자 vs 범위 (항상 범위 권장)
   - 낙관/기본/보수 시나리오 3개로 전달

## 쓸 수 있는 방법론

- [[시계열 지표 예측]]: 과거 패턴 기반 (트렌드+계절성)
- [[Markov Chain 지표 예측]]: 유저 상태 기반 (리텐션 구조가 있을 때)
- [[LTV 예측]]: 코호트 기반 미래 가치 추정
- [[CausalImpact]]: 특정 이벤트의 효과 사후 측정

## 전달 방식

```
"다음 분기 MAU 예측"
→ 기본 시나리오: X (현재 트렌드 지속)
→ 낙관 시나리오: Y (캠페인 효과 + 계절성)
→ 보수 시나리오: Z (이탈률 상승 가정)
→ 주요 불확실성: [변수 명시]
```

## 피해야 할 함정

- 단일 숫자만 주는 것 → 틀렸을 때 책임 소재 문제
- "예측"을 "목표"로 전달하는 것 → 목적 혼동
- 과거 데이터 부족한데 정밀 예측하는 것 → 과신
- 예측 근거(방법론, 가정) 없이 숫자만 주는 것
