## 이 상황은 언제 오는가

- 대시보드에서 핵심 지표(매출, 전환율, DAU 등)가 전일/전주 대비 눈에 띄게 하락
- 스테이크홀더가 "왜 떨어졌냐"는 질문을 들고 옴
- 릴리즈 직후 / 이벤트 종료 직후 / 특정 날짜 이후

## 제일 먼저 할 질문

1. **데이터 파이프라인 문제인가?** — 로깅 누락, 집계 버그, ETL 지연
   - 원시 데이터 이벤트 카운트와 집계 테이블 비교
   - 어제/그제 데이터가 정상적으로 들어오고 있는가

2. **특정 세그먼트만 떨어졌는가?** — 플랫폼, 지역, 코호트, 채널별 분해
   - 전체 하락 = 광범위한 원인 (서버, 글로벌 이슈)
   - 일부 하락 = 특정 변경사항, 특정 집단 이탈

3. **이 시점에 어떤 변화가 있었는가?**
   - 제품 릴리즈, 마케팅 캠페인 시작/종료, 가격 변경, 외부 이벤트

4. **계절성/주기성인가?** — 전년 동기 대비, 전전주 동요일 대비 비교

## 쓸 수 있는 방법론

- **Decomposition 분해**: 지표 = 분자/분모로 쪼개어 어느 쪽이 원인인지 파악
- [[DiD (Difference-in-Differences)]]: 특정 집단에만 처치가 있었던 경우 처치 효과 추정
- **기여도 분석**: 세그먼트별 기여도 변화로 하락 주범 찾기
- [[지표 이상 탐지 방법론]]: 이게 통계적으로 유의한 하락인지 판단

## 피해야 할 함정

- 데이터 검증 없이 바로 원인 분석으로 뛰어드는 것
- 가장 눈에 보이는 원인(최근 릴리즈)에 바로 귀결하는 것
- 단일 지표만 보고 전체 맥락 무시 (다른 지표는 정상인가?)
- "하락이 맞다"고 확인 전에 보고하는 것
