## 한 줄 정의

고객 한 명이 서비스와의 관계 전체에서 만들어낼 것으로 기대되는 총 가치. 마케팅 투자 기준, 고객 세그먼트 우선순위 결정의 핵심 지표.

## 언제 쓰는가 (전제 조건)

- "이 고객 획득에 얼마까지 쓸 수 있는가" (CAC 상한선 계산)
- 고객 세그먼트별 가치 차이를 정량화하고 싶을 때
- 이탈 예측과 결합해 리텐션 캠페인 대상 우선순위 결정

## 핵심 접근법

### 1. 단순 코호트 기반 LTV
```
LTV = ARPU × 평균 수명(개월)
    = ARPU / 월간 이탈률
```
- 빠르고 직관적, 초기 추정에 유용
- 한계: 개인 이질성 무시, 이탈률이 코호트마다 다를 수 있음

### 2. Markov Chain 기반 LTV
- 유저 상태 전이 확률로 기대 잔존 기간 계산
- 흡수 상태(이탈)까지의 기대 전이 횟수 = 기대 수명
- 세그먼트별 전이 확률 적용 가능 → [[Markov Chain 지표 예측]]

### 3. BG/NBD + Gamma-Gamma 모델 (비계약 구조)
- BG/NBD: 구매 빈도와 이탈 확률을 동시에 모델링
- Gamma-Gamma: 구매당 금액 분포 모델링
- e-commerce처럼 구독 없이 반복 구매하는 구조에 적합
- `lifetimes` Python 라이브러리로 구현 가능

### 4. 머신러닝 기반
- 피처: 최근성(Recency), 빈도(Frequency), 금액(Monetary) + 행동 피처
- 타겟: 향후 N개월 누적 매출
- 장점: 복잡한 비선형 패턴 포착 / 단점: 해석 어려움, 새 유저에 약함

## 핵심 함정

1. **LTV > CAC 확인만으로 부족**: 시간 가치 무시 (1년 후 LTV와 지금 CAC를 직접 비교하면 안 됨) → 할인율 적용한 **DLTV(Discounted LTV)** 사용
2. **이탈률 과소평가**: 초기 코호트는 헤비유저 비율이 높아 이탈률이 낮게 보임 → 성숙 코호트 기준으로 추정
3. **세그먼트 혼용**: 전체 평균 LTV로 의사결정하면 고가치 세그먼트를 과소평가, 저가치를 과대평가
4. **예측 LTV를 현실 LTV로 착각**: LTV는 추정값, 항상 불확실성 구간 포함
