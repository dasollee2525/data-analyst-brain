## 한 줄 정의

유저나 지표를 유한한 **상태(state)** 집합으로 정의하고, 상태 간 전이 확률로 미래를 예측하는 방법. "지금 상태"만 알면 다음 상태 예측이 가능하다는 Markov 가정이 핵심.

## 언제 쓰는가 (전제 조건)

- 유저/지표를 이산적인 상태로 나눌 수 있을 때
- 상태 전이 패턴이 시간에 걸쳐 비교적 안정적일 때
- 코호트 단위로 미래 지표 흐름을 예측하고 싶을 때

**DA 실무 적용 예시**
- 유저 상태: `신규 → 활성 → 휴면 → 이탈 → 재활성`
- 구독 상태: `무료 → 트라이얼 → 유료 → 해지`
- 퍼널 상태: `방문 → 가입 → 첫구매 → 재구매`

## 핵심 개념

**전이 행렬 (Transition Matrix)**
```
           활성   휴면   이탈
활성  [ 0.70  0.20  0.10 ]
휴면  [ 0.15  0.60  0.25 ]
이탈  [ 0.05  0.10  0.85 ]
```
- 각 행의 합 = 1
- n기 후 상태 분포 = 초기 분포 × 전이행렬^n

**정상 분포 (Stationary Distribution)**
- 전이행렬을 반복 곱하면 수렴하는 장기 상태 비율
- "이 제품의 유저 구조가 장기적으로 어떻게 될 것인가"를 보여줌

## 실무 활용 패턴

1. **다음 분기 활성 유저 수 예측**: 현재 코호트 상태 분포 × 전이행렬
2. **이탈 위험 유저 식별**: 휴면 상태 유저 중 이탈 전이 확률이 높은 세그먼트
3. **리텐션 캠페인 시뮬레이션**: 캠페인이 전이 확률을 바꾼다면 → 새 전이행렬로 재계산
4. **LTV 근사**: 흡수 마르코프 체인(이탈이 흡수 상태)으로 기대 활성 기간 계산

## 핵심 함정

1. **Markov 가정 위반**: 실제 유저 행동은 과거 이력에 영향을 받음 (예: 2번 이탈한 유저는 복귀 확률이 다름) → Higher-order Markov 또는 세그먼트별 전이행렬 필요
2. **전이 확률 안정성 가정**: 마케팅 캠페인, 시즌, 제품 변화로 전이 확률이 바뀌면 예측 빗나감 → 주기적 재추정 필요
3. **상태 정의의 자의성**: 활성/휴면 기준을 어떻게 잡느냐에 따라 결과가 크게 달라짐
4. **집계 오류**: 개인 수준 전이를 집계해 코호트 전이 확률로 쓸 때 Simpson's paradox 주의
