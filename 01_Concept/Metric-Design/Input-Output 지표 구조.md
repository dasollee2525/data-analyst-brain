## 한 줄 정의

Output 지표(결과)는 팀이 직접 움직이기 어렵다. Input 지표(행동)를 움직여 Output을 바꾸는 인과 사슬을 설계하는 프레임.

## 언제 쓰는가 (전제 조건)

- OKR/목표 지표만 있고 "어떤 행동을 해야 하는가"가 불분명할 때
- 팀이 지표를 보고 있지만 실제로 아무것도 못 하고 있을 때
- North Star Metric이 너무 lagging이라 주간 판단에 쓸 수 없을 때

## 구조

```
Output (결과 지표)         ← 우리가 만들려는 가치, 움직임이 느림
    ↑
Input (행동 지표)          ← 팀이 직접 통제 가능, 빠르게 반응
    ↑
Leading Indicator          ← Output의 선행 신호
```

**예시 (e-commerce)**
```
Output:  월간 매출
   ↑
Input:   장바구니 추가율, 결제 완료율, 재구매율
   ↑
Leading: 상품 상세페이지 체류시간, 리뷰 열람율
```

**좋은 Input 지표의 조건**
1. 팀이 직접 행동으로 움직일 수 있음
2. Output과 인과관계가 있음 (상관관계만으로 부족)
3. 빠르게 측정 가능 (주간 or 일간)
4. 조작하기 어려움 (게이밍 방지)

## 핵심 함정

1. **인과 없는 상관**: Input이 Output과 상관은 있지만 인과가 없는 경우 → Input을 올려도 Output이 안 움직임
2. **Input 과다**: Input 지표가 너무 많으면 팀이 무엇에 집중해야 할지 모름 → 1~3개로 제한
3. **Metric farming**: Input 지표 자체를 최적화하다가 Output이 오히려 나빠지는 경우
   - 예: 장바구니 추가율을 올리려고 강제 추가 UX → 결제 완료율 하락
4. **지표 계층 불일치**: 팀별 Input 지표가 회사 Output 지표와 연결이 안 되는 경우
